# Contributing to Consistency Auditor

Thank you for your interest in contributing! We welcome pull requests, bug reports, and feature requests.

## Development Process

1.  **Fork** the repository.
2.  **Clone** your fork locally.
3.  Create a **new branch** for your feature or bugfix (`git checkout -b feature/amazing-feature`).
4.  Run **tests** locally to ensure everything works (`make test`).
5.  Commit your changes with clear messages.
6.  Push to your fork and submit a **Pull Request**.

## Coding Standards

*   We use standard Go formatting (`gofmt`).
*   Please run `make lint` before submitting.
*   Add comments for exported functions and complex logic.

## Reporting Bugs

Please include:
*   Steps to reproduce the issue.
*   Expected vs. actual behavior.
*   Logs or error messages.
